Syntax

Variables

Variables are declared like this:

nmbr thisIsANumber = 2.5

First, the variable, then, the name and finally, the equal sign and value

List of allowed variables:

nmbr thisIsANumber = 2.5

strng thisIsAString = "Hello World!"

bool thisIsABoolean = true

vec2 thisIsAVector2 = {2.5, 2.5}

vec3 thisIsAVector3 = {2.5, 2.5, 2.5}

Functions

Functions in Float Code are defined like this:

fn nmbr numberFunction(nmbr arg1) = {}

'fn' for saying it's a function, then the return type of the function, then name and arguments

It's used a 'return' keyword for the return value, E.G.:

fn nmbr numberFunction(nmbr arg1) = {

return 2.5

}


The return value has to be of the type of the return type declared:

fn nmbr numberFunction(nmbr arg1) = {

return "a string"

}

This would return an error

If Else Statements

If Else Statements are declared like this:

if condition == true {

strng code = "Code"

}

else if otherCondition == true {

strng otherCode = "Other Code"

}

else {

strng lastCode = "Last Code"

}

For, Foreach and While Loops

While, For and Foreach Loops are very similar, but they are declared differently:

forEach a in b {

}


for i..bSize {

nmbr a = b[i]

}


nmbr i = 0

while i <= bSize {

nmbr a = b[i]

i++

}

All do the same, loop through every item 'a' in the array 'b'

Comments

Comments can be single-line and multi-line:

# single-line comment

#-

multi-line comment

-#

Semicolons

Semicolons in most cases are optional:

strng works = "Without"

strng alsoWorks = "With";

But in some cases, they are necessary:

nmbr thisWorks = 2.5; strng withThe = "semicolon"

nmbr DoesntWork = 2.5 strng withoutThe = "semicolon"

This would give an error because of the lack of a semicolon
